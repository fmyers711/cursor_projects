The challenge public-sector IT faces today

State agencies, local governments, and higher education institutions generate vast volumes of machine data—security events, application logs, infrastructure metrics, research telemetry, and compliance evidence. Yet most organizations face an impossible choice:

  • Expensive indexing — Index everything in Splunk and blow through constrained budgets.
  • Dropping data — Delete or sample telemetry, creating blind spots for security, operations, and AI.
  • Unmanaged external lakes — Dump data into S3 or cloud storage with no governance, discoverability, or path to analytics.

The result: more than 90% of operational data lands in low-cost cloud storage—unstructured, inaccessible, and untapped for AI and ML. For SLED environments, this is compounded by tight budgets, lean IT and SOC teams, fragmented agency or campus silos, and long-term compliance mandates that demand retention without proportional staffing.

The solution — Cisco Data Fabric

CDF unifies the full machine-data lifecycle under one governed architecture. Instead of choosing between cost and coverage, organizations can land everything economically, discover and understand data in place, promote only high-value slices into analytics or hot tiers, and search or feed AI across all tiers and external sources through federated search.

Three-tier architecture

CDF organizes machine data into three retention and capability bands. Each tier maps to named Splunk products with clear use cases—so CIOs and CISOs can align spend to mission priority rather than paying hot-tier prices for archive data.

Tier A is provided by core Splunk Indexes and premium applications. Tiers B and C are provided by the Machine Data Lake (MDL).

Tier A — Hot Real-time operations Products: Splunk Index, Enterprise Security, Observability Cloud Retention: 7–14 days (security example) Use cases: CIM detections, active investigation, monitoring, troubleshooting
Tier B — Analytics Scaled analytics & ML Product: MDL Analytics Table Retention: 90–180 days Use cases: OCSF batch detections, enriched search, ML training, agentic investigation
Tier C — Archive Long-term evidence Products: MDL Raw Tier / Long-Term Retention Retention: Long-term Use cases: Compliance, audit, forensic lookback, historical IOC hunting, promotion on demand

Competitive landscape — Frontier AI access to MELT data

State AI programs, campus research labs, and SOC modernization initiatives are converging on the same requirement: frontier models need governed access to Metrics, Events, Logs, and Traces (MELT)—not generic documents, but environment-specific operational evidence. Most alternative approaches solve storage or chat, not the semantic access layer frontier models require. The result is a widening capability gap across efficiency, cost, and reasoning quality.

The core gap

Frontier AI models are powerful reasoners, but raw MELT is not AI-ready at enterprise scale. It is too voluminous, repetitive, context-poor, and semantically inconsistent to feed directly into agentic models—or into naive retrieval pipelines—without prohibitive token cost, latency, and hallucination risk.

Why raw MELT breaks AI economics

Machine telemetry carries structural properties that collide with frontier-model constraints: finite context windows, per-token pricing, multi-step reasoning loops, and the need for governed, auditable access in SLED environments.

Structural challenge Effect on AI efficiency Cost / scale impact
Massive volume Billions of events/day exceed any context window; agents must iterate in loops Each loop multiplies token spend and wall-clock latency
Repetitive structure 90%+ of tokens repeat boilerplate fields (timestamps, hostnames, sourcetypes, JSON keys) Paying frontier-model rates for noise, not signal

The raw RAG tax — why repetitive data is expensive twice

Raw RAG—embedding and retrieving unstructured log chunks without upstream curation—is the most common DIY pattern in SLED AI pilots. It fails on MELT for two compounding reasons:

  1. Indexing tax — Every near-duplicate event field (hostname, sourcetype, JSON envelope) is embedded and stored again. A 90-day security corpus can generate millions of chunks where >90% of token mass is boilerplate.
  2. Inference tax — Retrieved chunks are injected into frontier-model context at per-token rates. Repetitive structure means the model spends capacity parsing format, not reasoning over evidence.
Cost layer Raw RAG on MELT CDF / MDL curated access
Data landing Cheap storage—but unmanaged, no governance catalog Economic MDL landing with catalog, RBAC, and promotion paths
Retrieval prep Re-embed on ingest, schema change, or time-window shift Promote structured slices (Analytics Table, Parquet, vector indexes) once; query via SPL, APIs, or MCP
Per-investigation tokens ~1.5M–2M+ (raw events or chunky RAG hits) ~3K–15K (aggregated, promoted, or semantically enriched context)
Reasoning loops Many—model must correlate, filter, and disambiguate noisy input Fewer—search-in-place and SPL aggregation return summaries, not raw firehoses
Governance Hard to enforce CJIS, FERPA, or agency-scoped access on arbitrary chunks Controlled promotion, audit trails, and index-level RBAC before AI consumption

Formula buyers can understand

AI investigation cost ≈ (model calls × tokens per call × price per token) + human validation time

Raw RAG inflates both variables. CDF reduces tokens per call through promotion, aggregation, and open-format export—and reduces call count by giving agents higher-confidence, pre-discoverable context earlier.

Security workflow example

A typical SLED security operations flow through CDF tiers:

  1. Data intake — Routed via Edge Processor / Ingest Processor.
  2. Immediate operations (Tier A) — High-value CIM detections remain in Splunk Index (Hot) with Enterprise Security; 7–14 day retention for active investigation.
  3. Scaled analytics (Tier B) — Batch OCSF detections and extended investigations land in MDL Analytics Table; 90–180 day retention.
  4. Long-term evidence (Tier C) — Raw and long-tail evidence held in MDL Long-Term Retention; small raw scans and promotion back up when deeper investigation is needed.
  5. Agentic response (future) — Feeds Triage, Guided Response, and Malware Reversing agents as they become available.
